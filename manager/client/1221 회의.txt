- 예약 기능이 추가로 개발되었으며, 이후 메시지 발송 기능도 추가됨

- 현재 계약 금액은 1500만원이며, 개발팀은 이 금액 이상의 업무를 수행했다고 주장

### 커뮤니케이션 및 요구사항 명확화 문제

- 메시지 발송에 필요한 최소 요구사항 리스트가 명확히 전달되지 않음
- 클라이언트 측에서 구체적인 요구사항을 메일이나 슬랙으로 전달한 적이 없음
- 실무 개발자와의 직접 소통이 부족했으며, 대부분 대표와의 만남에서만 확인됨
- 양측 모두 커뮤니케이션 부족을 인정 - 클라이언트는 명확한 지시가 없었고, 개발팀은 더 적극적으로 질문하지 않음
- 클라이언트는 산출물 리스트를 먼저 확인 후 피드백을 주겠다고 했으나, 확인 후에야 누락을 지적

### 현재 개발 상태 및 기술적 이슈

- 메시지 발송 결과 조회 기능이 현재 스코프에 포함되는지에 대한 의견 차이
- 발송 결과 화면을 만들기 위한 데이터가 필요하나, 현재는 성공한 경우만 저장되고 있음
- 개발팀은 30분 내에 발송 결과 페이지를 만들 수 있다고 언급
- API 200, 400 응답 데이터를 이미 확인했으며, 저장하는 것은 간단한 작업
- 발송 결과 조회 API 추가 및 테이블 생성이 필요

### 산출물 및 문서화 이슈

- 산출물 내용이 많이 누락되어 있으며, 리스트도 일부 빠져 있음
- 리눅스 이관 방법, 마이그레이션, DB 설정 등에 대한 가이드가 부족
- 개발팀은 더 자세한 가이드를 제공할 수 있다고 함
- 산출물 리스트는 계약서나 초기 이메일에 명시되어 있을 것이라고 언급

### 일정 및 진행 상황

- 일정이 지연되면 검수 및 이관도 함께 미뤄질 것으로 예상
- 개발팀은 빠른 개발 완료를 위해 명확한 요구사항 전달을 요청

### UI/UX 구성

- 메시지 발송 페이지에 새로운 탭을 추가하여 발송 내역을 표시
- 의뢰 단위(발송 버튼 클릭 단위)로 내역을 보여주는 방식
