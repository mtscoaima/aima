# 프로젝트 명세: B2C/B2B 메시지 발송 웹포탈 구축 (10주 집중 개발 계획)

## 1. 프로젝트 개요

### 1.1. 프로젝트명
- 메시지 발송 웹포탈 개발 (확장)

### 1.2. 프로젝트 목표
- 캠페인 중심의 사용자 친화적 메시지 발송 UI 및 포괄적인 관리 시스템 구축
    - PG 연동을 통한 결제 기능 구현
    - 정산 시스템 구축
    - 다단계 리워드 시스템 도입
- AI 기능 강화를 통한 사용자 편의성 및 마케팅 효율 증대
    - AI 기반 템플릿 추천 및 자동 생성
    - AI 기반 캠페인 타겟팅 자동 추천
    - AI 기반 사용자 행동 분석 및 캠페인 성과 예측
- 추가 메시징 채널 지원 확장을 통한 마케팅 채널 다변화
    - RCS (Rich Communication Services)
    - 카카오 알림톡
    - 카카오 친구톡플러스

### 1.3. 개발 기간
- 총 10주 (모든 기능 포함, 압축 개발 일정)
    - *참고: 원 견적서 상 1차 10주 (기존 기능 + AI 강화) + 2차 4주 (추가 채널) 구성이었으나, 본 계획은 10주 내 모든 범위 완료를 목표로 함.*

## 2. 기술 스택 및 개발 환경

### 2.1. 프론트엔드
- **언어/프레임워크**: Next.js (v13+ App Router), TypeScript, React (v18+)
- **상태 관리**: Zustand 또는 Recoil (논의 후 결정)
- **UI 라이브러리**: Tailwind CSS, Headless UI (또는 Shadcn/ui 고려)
- **데이터 페칭**: SWR 또는 React Query
- **폼 관리**: React Hook Form
- **기타**: ESLint, Prettier, Axios

### 2.2. 백엔드/데이터베이스
- **플랫폼**: Firebase
    - **데이터베이스**: Firestore (NoSQL Document Database)
    - **인증**: Firebase Authentication (Email/Password, Social Login 고려)
    - **서버리스 함수**: Firebase Functions (TypeScript 기반)
    - **스토리지**: Firebase Storage (이미지, 파일 업로드용)

### 2.3. 인프라
- **호스팅**: Firebase Hosting
- **CI/CD**: GitHub Actions (Lint, Test, Build, Deploy 자동화)

### 2.4. AI 서비스 연동
- **LLM**: OpenAI API (GPT-3.5/4, 필요시 클라이언트 계정 활용)
- **이미지 생성**: OpenAI DALL-E API 또는 Stability AI API (논의 후 결정)

### 2.5. 협업 도구
- **버전 관리**: Git, GitHub
- **커뮤니케이션**: Slack, Google Meet (또는 클라이언트 지정 채널)
- **문서화**: Notion 또는 Google Docs (API 명세는 Swagger/OpenAPI 사용)
- **디자인 협업**: Figma

## 3. 개발 범위 상세

### 3.1. 기획 및 설계 (1주차 집중)
- PC/MO 반응형 웹 전체 흐름 정의 및 UX 시나리오 구체화
- 화면 플로우 및 와이어프레임 상세 설계 (Figma 기반)
- Firestore 데이터베이스 스키마 상세 설계 (컬렉션, 문서 구조, 인덱싱 계획 포함)
- Firebase Functions 기반 API 엔드포인트 및 Request/Response 스펙 정의 (Swagger/OpenAPI 형식)
- AI 기능 (템플릿, 타겟팅) 및 확장 채널 연동 아키텍처 설계

### 3.2. UI/UX 디자인 (2주차 집중)
- 관리자, 사용자, 캠페인 관리 페이지 전체 시안 디자인 (PC/Mobile 반응형)
- 레퍼런스 사이트(`solapi.com`, `ppurio.com`, `findjob.co.kr`) 분석 기반 UI/UX 최적화
- 템플릿 그래픽 에디터 UI/UX 디자인:
    - 텍스트 편집: 폰트, 크기, 색상, 정렬, 위치 조정
    - 이미지 핸들링: 업로드, 크기 조절, 위치 조정
    - 레이어 관리 개념 도입 고려
- AI 템플릿 생성 및 캠페인 자동화 관련 UI/UX 디자인
    - 추천 템플릿 표시, 자동 완성 인터페이스, 타겟 그룹 추천 시각화

### 3.3. 프론트엔드 개발
- **공통**:
    - Next.js + TypeScript 프로젝트 구성
    - 반응형 디자인 시스템 구축 (Tailwind CSS 기반)
    - 공통 컴포넌트 라이브러리 개발
- **사용자 기능**:
    - 회원가입 (이메일, 소셜), 로그인, 마이페이지 (정보 수정, 비밀번호 변경)
    - 사용자 대시보드 (발송 현황, 크레딧 잔액, 주요 공지 등)
    - 캠페인 등록: 명칭, 목표 설정, 예산 설정, 기간 설정
    - 메시지 내용 작성: 채널별(SMS, LMS, MMS, RCS, 알림톡, 친구톡플러스) 템플릿 선택 및 작성
    - 그래픽 에디터 구현: (5주차)
        - 텍스트 및 이미지 편집 기능

    - 대상자 업로드: CSV, Excel 파일 파싱 및 유효성 검증, 직접 입력, 그룹 관리
    - 발송 설정: 즉시 발송, 예약 발송, 반복 발송
    - AI 템플릿 추천 및 자동완성 UI 연동 (5주차)
    - 채널 간 템플릿 자동 연동 기능 (한 채널 템플릿 작성 시 타 채널 자동 적용 UI) (8주차)
- **관리자 기능**:
    - 관리자 전용 인증 및 인터페이스
    - 사용자 관리: 목록 조회, 정보 수정, 상태 변경, 권한 제어
    - 캠페인 관리: 목록 조회, 상세 보기, 승인/반려 처리
    - 발신번호 검수 및 승인 기능 (증빙 서류 업로드 및 확인)
    - 기업 서류 검수 및 승인 기능
    - 메시지 발송 이력 및 통계 (채널별, 기간별, 성공/실패 등)
    - AI 생성 템플릿 및 캠페인 검수 기능
    - PG 연동 현황, 정산 관리, 리워드 지급 관리
    - 시스템 설정 (공지사항 관리, 약관 관리 등)

### 3.4. 백엔드 개발 (Firebase Functions)
- **공통**:
    - RESTful API 설계 원칙 준수
    - Firebase Admin SDK (TypeScript) 사용
    - 에러 핸들링 및 로깅 (Firebase Logging)
- **핵심 로직**:
    - 사용자 인증 및 세션 관리 (Firebase Authentication 연계)
    - Firestore CRUD 로직 (사용자, 캠페인, 메시지, 템플릿, 대상자 그룹 등)
    - 메시지 발송 요청 처리:
        - *중요: 실제 메시지 발송은 의뢰인 측 제공 메시지 서버와 인터페이스 테이블(DB) 적재 방식으로 진행. 본 프로젝트는 해당 테이블에 데이터 적재까지만 책임.* (세부 인터페이스 스펙 필요)
    - 파일 업로드 처리 (Firebase Storage 연동, 대상자 파일, 증빙 서류 등)
- **관리자 API**:
    - 사용자, 캠페인, 발신번호, 기업서류 승인/검토 로직
    - 통계 데이터 집계 및 제공 API
- **AI 연동 API**: (OpenAI API 등 외부 서비스 호출)
    - 템플릿 자동 생성 요청 및 결과 처리
    - 템플릿 추천 로직 (사용자 정보, 캠페인 정보 기반)
    - 캠페인 타겟팅 자동 추천 로직
    - 캠페인 성과 예측 및 최적화 추천 로직 (데이터 수집 후 단계적 구현)
- **확장 메시징 채널 연동 API**: (RCS, 알림톡, 친구톡플러스)
    - 각 채널별 발송 요청 API (메시지 서버 인터페이스 테이블 연동)
    - 채널별 템플릿 관리 및 동기화 로직
- **PG 및 정산 연동**:
    - 카드 결제 요청 처리 (PG사 API 연동, Firebase Functions 보안 규칙 철저히)
    - 결제 성공/실패 결과 처리 및 Firestore 업데이트
    - 정산 데이터 생성 및 관리 로직
    - 세금계산서 발행용 데이터 생성
- **영업사원 리워드 시스템 (다단계)**:
    - 추천인 코드 생성 및 관리
    - 고객 유입 경로 추적 (추천인 기반)
    - 다단계 MLM 구조 관리 (Firestore 데이터 모델링)
    - 직접/간접 추천에 따른 차등 수수료 계산 로직
    - 계층별 수수료 비율 설정 및 관리 기능 (관리자)
    - 매출 연동 인센티브 자동 계산
    - 수수료 지급 주기 설정 및 자동 정산 기능 (정산 데이터 생성)
    - 영업사원별 실적, 수익, 조직도 데이터 제공 API

### 3.5. 데이터베이스 설계 (Firestore)
- **주요 컬렉션**:
    - `users`: 사용자 정보 (인증 ID, 역할, 크레딧, 추천인 정보 등)
    - `campaigns`: 캠페인 정보 (명칭, 목표, 예산, 기간, 상태, 생성자 등)
    - `messages`: 개별 메시지 정보 (캠페인 ID, 수신자, 내용, 채널, 발송 상태, 발송 시간 등)
    - `templates`: 메시지 템플릿 (채널별, 내용, AI 생성 여부, 그래픽 요소 포함 시 관련 정보)
    - `targetGroups`: 대상자 그룹 정보 (그룹명, 대상자 목록/파일 경로)
    - `payments`: 결제 정보 (사용자 ID, 금액, PG사 거래 ID, 상태)
    - `settlements`: 정산 정보 (기간, 금액, 상태)
    - `rewards_config`: 리워드 정책 설정 (레벨별 수수료율 등)
    - `rewards_users`: 영업사원 정보 (추천 트리 구조, 수익 정보)
    - `rewards_history`: 리워드 발생 내역
    - `approvalRequests`: 발신번호/기업서류 승인 요청
    - `system_logs`: 주요 시스템 이벤트 로깅
- **인덱싱 전략**: 주요 조회 쿼리에 맞춰 복합 인덱스 생성 계획
- **보안 규칙**: 사용자 역할 및 데이터 소유권에 따른 엄격한 접근 제어 설정

### 3.6. AI 기능 강화
- **회원정보 기반 템플릿 자동 추천**: 사용자 프로필, 과거 캠페인 이력 등을 분석하여 적합한 템플릿 추천
- **회원정보 기반 캠페인 타겟팅 자동 추천**: 유사 사용자 그룹 또는 특정 행동 패턴을 보이는 사용자 그룹 추천
- **템플릿 그래픽 에디터와 AI 이미지 생성 통합**: 에디터 내에서 텍스트 입력 시 관련 이미지 AI 생성 및 삽입 제안
- **채널 간 템플릿 자동 동기화**: 특정 채널용으로 작성된 템플릿을 다른 채널 형식에 맞게 AI가 자동 변환/조정 제안
- **사용자 행동 데이터 수집 및 분석**: 클릭률, 전환율 등 주요 지표 수집 및 AI 모델 학습 데이터로 활용 (AI 고도화 기반)
- **캠페인 성과 예측 및 최적화 추천**: 과거 데이터 기반으로 캠페인 성공 가능성 예측 및 개선 방안 제안

### 3.7. 추가 메시징 채널 지원
- **RCS**: 발송, 결과 수신, 템플릿 관리 (이미지, 버튼 등 RCS 특화 기능 포함)
- **카카오 알림톡**: 카카오 채널 연동, 템플릿 등록 및 검수 프로세스 연동, 발송, 결과 수신
- **카카오 친구톡플러스**: 이미지, 와이드형 메시지 등 지원, 발송, 결과 수신
- **통합 관리**: 채널별 템플릿 통합 관리, 통합 발송 대시보드 및 통계 제공

## 6. 특이사항 및 유의점

- **메시지 서버 연동**: 실제 SMS/MMS/RCS 등 발송은 클라이언트 측에서 제공하는 메시지 서버의 인터페이스 테이블(DB)에 데이터를 적재하는 방식으로 진행. 해당 DB 스펙 및 연동 방식에 대한 명확한 정의가 프로젝트 초기에 필요.
- **외부 시스템 연동**: PG사, 카카오 알림톡/친구톡플러스 API 등 외부 시스템 연동 시 클라이언트 측의 사전 계약 및 기술 지원 필요. API Key 등 연동 정보는 안전하게 관리.
- **디자인 리소스**: 로고, 브랜드 컬러 등 핵심 디자인 요소는 클라이언트 제공 전제.
- **AI 서비스 연동 비용**: OpenAI API 등 외부 AI 서비스 사용에 따른 비용은 별도로 발생할 수 있으며, 클라이언트 계정으로 연동하는 것을 기본으로 함. 사용량 제한 및 비용 최적화 방안 고려.
- **영업사원 리워드 시스템 복잡도**: 다단계(MLM) 구조의 리워드 시스템은 로직이 복잡할 수 있으므로, 정책 확정 및 예외 케이스 처리에 대한 충분한 논의 필요.
- **10주 압축 일정**: 원 견적서의 14주 분량을 10주로 압축 진행하므로, 각 주차별 목표 달성을 위한 집중적인 노력과 효율적인 의사소통이 매우 중요. 범위 변경 시 일정에 큰 영향을 줄 수 있음.

4.  **배포 가이드**: 프로덕션 환경 빌드, 배포, 롤백 절차 및 Firebase 프로젝트 설정 가이드
