# 📌 B2C/B2B 메시지 발송 웹포탈 구축 프로젝트 견적서

## ✅ 프로젝트 개요
* **프로젝트명**: 메시지 발송 웹포탈 개발
* **목표**: 캠페인 중심 메시지 발송 UI 및 관리 시스템 구축 (PG, 정산, 리워드 포함)
* **개발 기간**: 총 10주
* **개발 비용**: 10,000,000원 (부가세 별도)

---

## 🗂️ 주요 개발 범위

### 1. 기획 및 설계
* PC/MO 반응형 웹 흐름 정의
* 화면 플로우 및 와이어프레임 설계
* ERD 및 API 스펙 기획

### 2. UI/UX 디자인
* 관리자 / 사용자 / 캠페인 관리 페이지 디자인
* 반응형 웹 기준 (PC/MO 대응)

### 3. 프론트엔드
* Preact 또는 Vue.js + TypeScript 기반 SPA 개발
* 캠페인 등록, 메시지 내용 작성, 대상자 업로드
* 회원가입 및 인증, 관리자 전용 기능 구현

### 4. 백엔드
* Spring Boot + PostgreSQL + Java 기반 API 서버 개발
* 사용자 / 캠페인 / 결제 / 리워드 / 정산 로직 구현
* 메시지 발송 요청은 인터페이스 테이블(DB) 적재만 진행 (별도 메시징 API 연동 없음)

### 5. PG 및 정산 연동
* 카드 결제 연동 (PG사 연동)
* 영업사원 포인트/리워드 적립 로직 구현
* 세금계산서 발행용 자료 생성 기능

### 6. 관리자 기능
* 이력/로그 조회 및 필터링
* 사용자 및 캠페인 관리
* 정산, 리워드 통계 확인

## 💬 관리 및 협업 방식
* **주단위 산출물 실시간 공유**: 매주 진행상황 및 산출물을 실시간으로 공유하여 투명한 커뮤니케이션 유지
* **소통 채널**: 이메일, 카카오톡, Zoom 등을 통한 신속한 커뮤니케이션
* **정기 보고**: 주 1회 진행상황 보고 및 이슈 논의
* **피드백 반영**: 클라이언트 피드백을 신속하게 개발에 반영

---

## 📎 특이사항 및 유의점
* 메시지 발송은 실제 API 연동이 아닌, DB 테이블 적재만 진행
* 외부 시스템 연동은 클라이언트 제공 스펙 기준으로 대응
* 디자인 리소스(로고, 브랜드 컬러 등) 클라이언트 제공 전제
* PG사는 클라이언트 사전 계약 완료되어야 연동 가능
* 레퍼런스 사이트(solapi.com) 참고하여 UI/UX 구성
