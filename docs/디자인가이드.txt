# 동글동글한 디자인 가이드

## 기본 디자인 원칙
- 모든 UI 요소는 둥근 모서리 처리 (border-radius: 12px~16px)
- 부드러운 색상 팔레트 사용 (파란색 계열 중심, 포인트 컬러 적절히 활용)
- 간결하고 직관적인 사용자 경험 제공
- 여백을 충분히 활용한 시원한 레이아웃

## 컴포넌트별 가이드
 
### 버튼
- 모든 버튼은 라운드 처리 (border-radius: 8px~12px)
- 주요 버튼: 파란색 배경, 흰색 텍스트
- 보조 버튼: 흰색 배경, 테두리 처리, 파란색 텍스트
- 호버 시 약간의 그림자 효과 추가

### 카드
- 모든 카드는 둥근 모서리 (border-radius: 12px)
- 가벼운 그림자 효과 (box-shadow: 0 2px 8px rgba(0,0,0,0.05))
- 카드 내부 여백 충분히 확보 (padding: 16px~24px)
- 호버 시 미세한 확대 또는 그림자 강화

### 아이콘
- 라인 아이콘 사용 (두께 2px)
- 아이콘 모서리 라운드 처리
- 컨텍스트에 맞는 의미 있는 아이콘 선택
- 크기 일관성 유지 (24px x 24px 기본)

### 색상
- 주요 색상: #0066FF (파란색 계열)
- 보조 색상: #F5F7FA (배경), #E5E8EC (구분선)
- 텍스트 색상: #333333 (기본), #666666 (보조), #999999 (약한 강조)
- 포인트 색상: #FF5500 (주황), #00C853 (초록)

### 타이포그래피
- 본문: 16px, 행간 1.5
- 제목: 20px~24px, 볼드
- 작은 텍스트: 14px
- 폰트 패밀리: Noto Sans KR, San-serif

### 네비게이션
- 상단 메뉴: 둥근 하단 테두리
- 드롭다운 메뉴: 라운드 처리된 박스
- 활성 메뉴 강조: 배경색 또는 밑줄로 표시

### 반응형 디자인
- 모바일에서도 동글동글한 느낌 유지
- 터치 영역 충분히 확보 (최소 44px x 44px)
- 모바일에서 단일 컬럼 레이아웃으로 전환

## 구현 가이드
- CSS 변수로 공통 값 관리 (색상, 둥글기, 여백 등)
- 컴포넌트 재사용성 고려
- 접근성 표준 준수 (적절한 대비, 키보드 네비게이션)
- 애니메이션은 부드럽고 자연스럽게 (transition: 0.2s ease-in-out)
